1. Resumen Ejecutivo

La Fase 2 construye la capa de herramientas que permite a Jarvis y sus agentes interactuar con el mundo real. Sin estas tools, los agentes solo pueden pensar — no actuar. Esta guía contiene todo lo necesario para implementarla en Antigravity desde cero.

¿POR QUÉ ES CRÍTICA LA FASE 2?

Sin herramientas, Jarvis es solo un chatbot. Con herramientas, es una oficina autónoma. Esta fase convierte el sistema en un agente capaz de navegar webs, enviar emails, ejecutar código y gestionar archivos sin intervención humana.

Orden de Implementación

Implementar en este orden estricto — cada tool valida que la infraestructura base funciona antes de añadir complejidad:

  1. API Gateway — Base para todas las demás tools
  2. Web Search — Valida conectividad externa
  3. File Manager — Valida conexión con Supabase Storage
  4. Code Executor — Valida sandbox de seguridad
  5. Web Browser — Valida Playwright y automatización
  6. Data Analyzer — Valida procesamiento de datos
  7. Email Manager — Valida OAuth y credenciales
  8. Communications — Valida webhooks y bots

Interfaz Unificada (TypeScript)

Todas las tools implementan esta interfaz. Sin excepción.

interface Tool {

name: string

description: string

input_schema: JSONSchema

execute(params: any): Promise<ToolResult>

log_to_supabase(result: ToolResult): Promise<void>

}

interface ToolResult {

success: boolean

data?: any

error?: string

tool_name: string

agent_id: string

duration_ms: number

tokens_used?: number

}

3. SQL Completo para Supabase

Ejecutar en Supabase > SQL Editor en este orden antes de implementar ninguna tool.

Tabla Maestra: tool_executions

-- Log de cada ejecución de cualquier tool por cualquier agente

CREATE TABLE tool_executions (

id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

tool_name TEXT NOT NULL,

agent_id TEXT NOT NULL,

agent_name TEXT,

input_params JSONB,

output_data JSONB,

status TEXT CHECK (status IN ('success','error','timeout')),

error_msg TEXT,

duration_ms INTEGER,

tokens_used INTEGER,

cost_usd DECIMAL(10,6),

created_at TIMESTAMPTZ DEFAULT NOW()

);

CREATE INDEX idx_tool_exec_agent ON tool_executions(agent_id);

CREATE INDEX idx_tool_exec_tool ON tool_executions(tool_name);

CREATE INDEX idx_tool_exec_date ON tool_executions(created_at DESC);

Tabla: files_registry (File Manager)

CREATE TABLE files_registry (

id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

filename TEXT NOT NULL,

path TEXT NOT NULL,

category TEXT, -- documentos|datos|reportes|codigo

size_bytes INTEGER,

mime_type TEXT,

storage_url TEXT,

modified_by TEXT, -- agent_id

metadata JSONB,

created_at TIMESTAMPTZ DEFAULT NOW(),

updated_at TIMESTAMPTZ DEFAULT NOW()

);

Tabla: search_cache (Web Search)

CREATE TABLE search_cache (

id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

query_hash TEXT UNIQUE NOT NULL,

query_text TEXT NOT NULL,

results_json JSONB NOT NULL,

source TEXT, -- serper|tavily|brave

result_count INTEGER,

expires_at TIMESTAMPTZ NOT NULL,

created_at TIMESTAMPTZ DEFAULT NOW()

);

CREATE INDEX idx_search_hash ON search_cache(query_hash);

CREATE INDEX idx_search_exp ON search_cache(expires_at);

Tabla: code_executions (Code Executor)

CREATE TABLE code_executions (

id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

agent_id TEXT NOT NULL,

language TEXT CHECK (language IN ('python','javascript')),

code_hash TEXT,

stdout TEXT,

stderr TEXT,

exit_code INTEGER,

duration_ms INTEGER,

timed_out BOOLEAN DEFAULT FALSE,

created_at TIMESTAMPTZ DEFAULT NOW()

);

Tabla: emails_log (Email Manager)

CREATE TABLE emails_log (

id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

agent_id TEXT,

direction TEXT CHECK (direction IN ('inbound','outbound')),

from_addr TEXT,

to_addr TEXT[],

subject TEXT,

body_preview TEXT,

classification TEXT, -- urgente|normal|spam|newsletter

action_taken TEXT,

thread_id TEXT,

has_attachments BOOLEAN DEFAULT FALSE,

created_at TIMESTAMPTZ DEFAULT NOW()

);

Tabla: notifications_log (Communications)

CREATE TABLE notifications_log (

id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

agent_id TEXT,

channel TEXT, -- whatsapp|telegram|slack|discord

direction TEXT CHECK (direction IN ('sent','received')),

to_address TEXT,

message TEXT,

status TEXT, -- sent|delivered|failed

metadata JSONB,

created_at TIMESTAMPTZ DEFAULT NOW()

);

4. Las 8 Tools — Prompts de Implementación

Cada sección contiene el prompt completo listo para pegar en Antigravity. Implementar una a una, verificando tests antes de continuar.

TOOL 1 — API Gateway

Infraestructura base

BASICA

KEYS REQUERIDAS

Ninguna

ESTADO

⬜ PENDIENTE

El API Gateway es la fundación. Todas las demás tools lo usan internamente para llamadas HTTP externas. Sin esta tool bien construida, el resto fallará.

PROMPT PARA ANTIGRAVITY — TOOL 1

Crea /office/tools/api-gateway/index.ts

Debe implementar un cliente HTTP universal con:

- Soporte para GET, POST, PUT, PATCH, DELETE

- Autenticación: API Key (header), Bearer Token, Basic Auth, OAuth2

- Rate limiting: máx N requests/minuto configurable por dominio

- Retry automático con backoff exponencial (3 intentos, 1s/2s/4s)

- Timeout configurable (default 30s)

- Logging automático en Supabase tabla tool_executions

- Gestión de credenciales desde tabla api_credentials (nunca hardcoded)

- Respuesta siempre tipada con ToolResult interface

- Manejo de errores estructurado (nunca crashear el proceso)

Exports requeridos:

apiCall(config: ApiCallConfig): Promise<ToolResult>

getCredential(integration_id: string, key: string): Promise<string>

Tests: crear api-gateway.test.ts con casos:

- GET exitoso a API pública

- Manejo de error 401, 429, 500

- Retry en error 503

- Timeout después de 30s

🔍

TOOL 2 — Web Search

Búsqueda en tiempo real

BASICA

KEYS REQUERIDAS

Serper API Key o Tavily API Key

ESTADO

⬜ PENDIENTE

Permite a los agentes buscar información actual en internet. Cache inteligente en Supabase para no repetir búsquedas idénticas y ahorrar costes de API.

PROMPT PARA ANTIGRAVITY — TOOL 2

Crea /office/tools/web-search/index.ts

Integrar con Serper API como primario y Tavily como fallback.

Funciones requeridas:

searchWeb(query, options?): Promise<SearchResult[]>

searchNews(query, options?): Promise<SearchResult[]>

searchImages(query, options?): Promise<ImageResult[]>

Lógica de cache:

1. Hash del query → buscar en tabla search_cache

2. Si existe y expires_at > NOW() → devolver cache

3. Si no → llamar API → guardar en cache (TTL: web=1h, news=15min)

Cada SearchResult debe tener:

{ title, url, snippet, source, published_at?, position }

Usar api-gateway internamente para todas las llamadas HTTP.

Variables de entorno: SERPER_API_KEY, TAVILY_API_KEY

📁

TOOL 4 — Code Executor

Ejecución segura de código

MEDIA

KEYS REQUERIDAS

Ninguna (sandbox interno)

ESTADO

⬜ PENDIENTE

Ejecuta Python y JavaScript en un sandbox aislado. Los agentes pueden generar y ejecutar código para resolver problemas sin acceso al sistema host.

⚠️ SEGURIDAD CRÍTICA

El sandbox NO debe tener acceso a: sistema de archivos del host, red externa, variables de entorno del sistema, ni procesos del sistema operativo.

PROMPT PARA ANTIGRAVITY — TOOL 4

Crea /office/tools/code-executor/index.ts

Implementar sandbox con vm2 (Node.js) para JavaScript

y subprocess con restricciones para Python.

Funciones:

executeCode(language, code, timeout?): Promise<ExecResult>

installDependency(language, package): Promise<void>

ExecResult: { stdout, stderr, exit_code, duration_ms, timed_out }

Restricciones de seguridad obligatorias:

- Timeout máximo 60s, default 30s

- Sin acceso a fs del host (solo /tmp/sandbox/)

- Sin acceso a red (excepto llamadas explícitas via api-gateway)

- Sin acceso a process.env del sistema

- Memoria máxima: 256MB

- Registrar en tabla code_executions

TOOL 5 — Web Browser

Navegación autónoma

AVANZADA

KEYS REQUERIDAS

Playwright (incluido en Antigravity)

ESTADO

⬜ PENDIENTE

Navegación web completamente autónoma. Los agentes pueden visitar páginas, extraer datos, rellenar formularios y capturar screenshots sin intervención humana.

PROMPT PARA ANTIGRAVITY — TOOL 5

Funciones requeridas:

visitPage(url): Promise<PageContent>

extractData(url, selector): Promise<any[]>

fillForm(url, fields): Promise<FormResult>

clickElement(selector): Promise<void>

screenshot(url, options?): Promise<Buffer>

scrapeStructured(url, schema): Promise<any>

PageContent: { title, text, links[], metadata, screenshot? }

Restricciones:

- Respetar robots.txt antes de hacer scraping

- Rate limit: máx 1 request/segundo por dominio

- Timeout de página: 30s

- Guardar screenshots en Supabase Storage via file-manager

- No acceder a dominios bloqueados (lista configurable)

Registrar todo en tabla notifications_log

5. Tool Registry — Jarvis conoce todo

Este archivo es el directorio central que Jarvis consulta para saber qué puede hacer y qué tool usar para cada tipo de tarea.

PROMPT PARA ANTIGRAVITY — TOOL REGISTRY

Crea /office/jarvis/tool-registry.ts

Importar todas las tools de /office/tools/index.ts

Crear un registro con la siguiente estructura por tool:

{

tool: ToolInstance,

triggers: string[], // palabras clave que activan esta tool

cost_estimate: 'free' | 'cheap' | 'medium' | 'expensive',

requires_credentials: string[], // IDs de api_credentials

can_be_parallelized: boolean,

timeout_ms: number

}

Crear función selectTool(task_description: string): Tool[]

Que analice la descripción de una tarea y devuelva las tools

más apropiadas ordenadas por relevancia y coste.

Crear función checkAllCredentials(): CredentialStatus[]

Que verifique cuáles tools tienen credenciales y cuáles no,

para mostrar en el Panel de Conexiones.
